[BAGIAN 1: WEATHER INTELLIGENCE (ENVIRONMENTAL SYNC)]

10.1. OpenWeatherMap API Integration
Sistem tidak hanya memantau air dari bawah (sensor), tetapi juga dari atas 
(langit) melalui integrasi satelit cuaca.

Snippet Logika Caching (`weather.blade.php`):
```javascript
const cacheKey = `sentinel_weather_${lat}_${lng}`;
if(Date.now() - data.ts < 600000) { // 10 Minute Cache
    applyWeatherData(data.weather);
    return;
}
```
**Analisis Teknis**: Untuk mengoptimalkan kuota API dan bandwidth, sistem 
menyimpan data cuaca di `localStorage` selama 10 menit. Ini memastikan 
dashboard tetap informatif tanpa melakukan request berulang yang tidak perlu.

10.2. Dynamic Dashboard Atmosphere (Glow Sync)
Fitur estetika paling canggih adalah perubahan warna atmosfer dashboard secara 
otomatis berdasarkan kondisi langit di lokasi sensor.
- **Hujan/Badai**: Dashboard berubah menjadi nuansa Indigo/Purple (`bg-purple-600/20`).
- **Cerah**: Dashboard berubah menjadi nuansa Amber/Gold (`bg-amber-400/20`).
Ini memberikan "Situational Awareness" instan kepada operator hanya dengan 
melihat warna pendaran layar.

[BAGIAN 2: HEARTBEAT MONITORING (HARDWARE VITALITY)]

10.3. Automated Offline Detection (`CheckDeviceHeartbeat.php`)
Sistem memiliki mekanisme "Watchdog" yang memantau kesehatan hardware setiap detik.

Snippet Logika:
```php
$threshold = now()->subSeconds(30);
$idleDevices = Device::where('status', 'online')->where('last_seen', '<', $threshold)->get();
```
**Alur Kerja**:
1.  **Polling**: Perintah `app:check-device-heartbeat` berjalan secara rutin.
2.  **Detection**: Jika perangkat tidak mengirim detak jantung (Heartbeat) 
    selama lebih dari 30 detik, status otomatis berubah menjadi `offline`.
3.  **Instant Update**: Sistem memicu event `DeviceStatusUpdated` via WebSocket, 
    menyebabkan marker di peta dashboard berubah menjadi merah SEKETIKA 
    tanpa perlu refresh.

[BAGIAN 3: THIRD-PARTY ECOSYSTEM (SSO & NOTIFICATIONS)]

10.4. Google SSO Integration
Menggunakan **Laravel Socialite**, sistem memungkinkan personil IT login 
menggunakan akun organisasi Google, meningkatkan keamanan melalui kebijakan 
2FA yang dikelola oleh Google.

10.5. WhatsApp Gateway Ready
Sistem dirancang dengan arsitektur yang siap dihubungkan ke API WhatsApp 
(seperti Fonnte atau Twilio). Logika pengiriman sudah terintegrasi di dalam 
`NotificationService`, siap untuk mengirimkan pesan darurat langsung ke HP 
warga jika radius banjir tercapai.

[BAGIAN 4: SCALABILITY & FUTURE ROADMAP]

10.6. Big Data Preparedness
Arsitektur database yang menggunakan **Partitioning Ready** dan **Tiered Storage** 
memungkinkan Sentinel WaterSense untuk menangani penambahan hingga 1.000+ node 
sensor di sepanjang aliran sungai Jawa Barat tanpa penurunan performa yang 
signifikan.

10.7. AI Flood Prediction (Roadmap)
Struktur data histori yang rapi di tabel `water_level_histories` telah 
disiapkan untuk menjadi *dataset* pelatihan model Machine Learning guna 
memprediksi datangnya banjir 6-12 jam sebelum air meluap berdasarkan korelasi 
data curah hujan dan TMA.
